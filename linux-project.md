# **AUX Project 1: Shell Scripting | Onboarding New Linux Users onto a Server**

## Introduction

In this project, 20 new users are created and onboarded onto a server. Each user is added to an existing group called developers. (The developer group would also be created in this project). The shell script would check the existence of a user before creation and also ensure each user has its own default home folder. Also each user would have a `.ssh` folder within its home folder. For each user's ssh configurarion, an authorized_key file would be created. A public and private key would be provided to be used for the project.

Two Ubuntu Vitrual Machines (VMs) are set up on the local machine. One is Ubutu desktop and the other is Ubuntu Sever. Both are of version 22.04. The desktop version would act as the client machine and the other as the server.

## Step 1: On the client machine

Change directory to the Desktop and create a folder named Shell

`cd Desktop/ && mkdir Shell && cd Shell`

In the Shell Directory, create the following files file "name.csv" and onboard.sh

`touch name.csv onboard.sh`

In the names.csv file, list names of the new users that would be onboarded to the server.

`vi name.csv`

![alt text](/img/users-scnsh.png)

Open Onboard.sh and write script to help carry out the objective of this project is written.

`nano onboard.sh`

```
#!/bin/bash
userfile=$(cat names.csv)
PASSWORD=password

# To ensure the user running this script has sudo privilege
    if [ $(id -u) -eq 0 ]; then

# Reading the CSV file
 for user in $userfile;
 do
            echo $user
        if id "$user" &>/dev/null
        then
            echo "User Exist"
        else

# This will create a new user
        useradd -m -d /home/$user -s /bin/bash -g developers $user
        echo "New User Created"
        echo


# This will create a ssh folder in the user home folder
        su - -c "mkdir ~/.ssh" $user
        echo ".ssh directory created for new user"
        echo

# We need to set the user permission for the ssh dir
         su - -c "chmod 700 ~/.ssh" $user
         echo "user permission for .ssh directory set"
         echo

# This will create an authorized-key file
        su - -c "touch ~/.ssh/authorized_keys" $user
        echo "Authorized Key File Created"
        echo

# We need to set permission for the key file
        su - -c "chmod 600 ~/.ssh/authorized_keys" $user
        echo "user permission for the Authorized Key File set"
        echo

# We need to create and set public key for users in the server
        cp -R "/home/sb/Shell/id_rsa.pub" "/home/$user/.ssh/authorized_keys"
        echo "Copied the Public Key to New User Account on the server"
        echo
        echo

        echo "USER CREATED"

# Generate a password.
sudo echo -e "$PASSWORD\n$PASSWORD" | sudo passwd "$user" 
sudo passwd -x 5 $user
            fi
        done
    else
    echo "Only Admin Can Onboard A User"
    fi
```

Replace the `/home/sb` in the script with you home user home directory path

Copy the Shell directory to your remote server.

`scp -r Shell remotename@remoteip:/home/remoteuser`

Note: Replace the remote details with the actual details.
